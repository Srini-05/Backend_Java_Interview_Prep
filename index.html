Java Basics

Fundamentals every Java developer must know cold

Q 01 What is Java? ▼
One-Line AnswerJava is a high-level, platform-independent, object-oriented programming language that runs on the JVM — famous for "Write Once, Run Anywhere."

What Makes Java Special?

Java was created by James Gosling at Sun Microsystems in 1995. The big idea was simple but revolutionary: write your code once, and it runs on any device that has a JVM (Java Virtual Machine). This is called platform independence.

When you write Java code, the compiler converts it into bytecode — not machine code. That bytecode is then run by the JVM on any platform (Windows, Linux, Mac, phone), making Java truly portable.

Key Features at a Glance

  • Platform Independent — Write Once, Run Anywhere (WORA)
  • Object-Oriented — Everything is organized around objects and classes
  • Automatic Memory Management — Garbage Collector handles memory for you
  • Strongly Typed — Type errors caught at compile time
  • Multithreaded — Built-in support for running tasks concurrently
  • Rich Ecosystem — Millions of libraries, Spring, Hibernate, Maven, etc.

Where Is Java Used?

  • 🏦 Enterprise Banking Applications (SWIFT, trading systems)
  • 📱 Android App Development
  • 🌐 Web Backends (Spring Boot REST APIs)
  • 📊 Big Data (Apache Hadoop, Spark)
  • ☁️ Cloud Applications (microservices)
In interviews, always mention WORA, garbage collection, and a real-world use case like banking or Android.

Q 03 What is JVM, JRE, and JDK? What is the difference? ▼
One-Line AnswerJVM executes bytecode, JRE = JVM + libraries to run apps, JDK = JRE + compiler + tools to build apps. They are nested: JDK ⊃ JRE ⊃ JVM.

Simple Analogy: The Kitchen

  • JVM = The stove — it actually does the cooking (executes code)
  • JRE = The kitchen — stove + utensils to run a recipe (run apps)
  • JDK = Kitchen + recipe book + chef's tools — to create and run recipes (develop apps)

Comparison Table

Key Tools Inside JDK

  • javac — Java compiler (converts .java → .class bytecode)
  • java — Launcher that starts JVM and runs your program
  • javadoc — Generates HTML documentation from comments
  • jar — Packages .class files into a JAR archive
  • jdb — Java debugger
Rule of thumb: You need JDK to write code, JRE to run code, and JVM is always running under the hood.

Q 05 What are access modifiers in Java? ▼
One-Line AnswerAccess modifiers control visibility: public (everywhere), protected (package + subclasses), default (same package), private (same class only).

When To Use Each?

  • private — Always for fields (encapsulation principle). Fields should never be directly public.

Q 06 What is the difference between == and equals()? ▼
One-Line Answer== checks if two references point to the same memory location. equals() checks if two objects have the same logical value/content.

Simple Memory Model

Think of it this way: two different houses can look identical (same content), but they are at different addresses (different references). == compares the address. equals() compares what's inside.

Comparison Table

Aspect==equals()
What it comparesMemory address / referenceLogical content / value
For primitivesCompares values directlyN/A (not applicable)
For objectsChecks if same instanceChecks if logically equal
Can be overridden❌ No✅ Yes (always override with hashCode)

Code Example

String s1 = new String("hello");
String s2 = new String("hello");
String s3 = s1; // same reference

System.out.println(s1 == s2);       // false — different objects in heap
System.out.println(s1.equals(s2));  // true  — same content "hello"
System.out.println(s1 == s3);       // true  — same reference

// String pool behavior
String a = "hello";  // from pool
String b = "hello";  // same pool object
System.out.println(a == b); // true! (both point to pool)
Always use equals() for String comparison! And whenever you override equals(), you MUST also override hashCode() — otherwise HashMaps and HashSets will break.

OOP Concepts

The four pillars and key object-oriented principles

Q 07 What is Inheritance? ▼
One-Line AnswerInheritance lets a child class acquire the properties and behaviours of a parent class using the extends keyword, promoting code reuse and IS-A relationships.

Real-World Analogy

A Dog IS-A Animal. A Dog inherits the ability to breathe, eat, and sleep from Animal — and adds its own behaviours like barking. You don't rewrite those common behaviours for every animal.

Code Example

class Animal {
    String name;
    void eat() { System.out.println("Animal eats"); }
    void breathe() { System.out.println("Animal breathes"); }
}

class Dog extends Animal {     // Dog IS-A Animal
    void bark() { System.out.println("Woof!"); }
}

Dog dog = new Dog();
dog.eat();    // inherited from Animal
dog.bark();   // Dog's own method

Types of Inheritance in Java

TypeSupported?Notes
Single✅ YesA extends B
Multilevel✅ YesA → B → C chain
Hierarchical✅ YesMany classes extend one parent
Multiple❌ No (classes)Achieved through interfaces
Hybrid❌ No (classes)Achieved through interfaces
Java doesn't allow multiple class inheritance to avoid the "Diamond Problem." Use interfaces to achieve it safely.
Q 08 What is Polymorphism? ▼
One-Line AnswerPolymorphism means "many forms" — the same method name behaves differently based on context. Achieved via overloading (compile-time) and overriding (runtime).

Two Types of Polymorphism

// 1. COMPILE-TIME (Method Overloading) — resolved at compile time
class Calculator {
    int    add(int a, int b)       { return a + b; }
    double add(double a, double b) { return a + b; }
    int    add(int a, int b, int c){ return a + b + c; }
}

// 2. RUNTIME (Method Overriding) — resolved at runtime
class Animal {
    void sound() { System.out.println("Generic sound"); }
}
class Dog extends Animal {
    @Override
    void sound() { System.out.println("Woof!"); }
}
Animal a = new Dog(); // reference type = Animal, actual type = Dog
a.sound(); // prints "Woof!" — runtime polymorphism

Compile-time vs Runtime Polymorphism

AspectCompile-time (Overloading)Runtime (Overriding)
Also calledStatic polymorphismDynamic polymorphism
Resolved when?At compile timeAt runtime
RequiresSame class, different paramsInheritance / @Override
PerformanceFasterSlight overhead (vtable lookup)
Polymorphism is the key to writing flexible, extensible code. A classic example is writing a method that accepts Animal type but works for Dog, Cat, Bird, etc.
Q 09 What is Abstraction? ▼
One-Line AnswerAbstraction hides complex internal details and exposes only the essential features. Achieved through abstract classes and interfaces.

Real-World Analogy

When you drive a car, you use the steering wheel and pedals. You don't need to know how the engine, fuel injection, or transmission work internally. That internal complexity is abstracted away.

// Abstract class: partial abstraction
abstract class Shape {
    abstract double area();    // subclass MUST implement
    void display() { System.out.println("Area: " + area()); }
}

class Circle extends Shape {
    double radius;
    Circle(double r) { this.radius = r; }
    @Override
    double area() { return Math.PI * radius * radius; }
}

// Interface: full abstraction (contract)
interface Printable {
    void print();  // all implementing classes must define this
}

Abstract Class vs Interface

FeatureAbstract ClassInterface
MethodsAbstract + concreteAbstract (+ default/static since Java 8)
VariablesAny typepublic static final only
Constructor✅ Yes❌ No
Multiple inheritance❌ No✅ Yes (implement many)
Use whenShared base behaviourDefining a contract
Use abstract classes when you have shared code. Use interfaces when you want to define a contract that many unrelated classes can follow.
Q 10 What is Encapsulation? ▼
One-Line AnswerEncapsulation wraps data (fields) and behavior (methods) together in a class, and hides internal state using private access + public getters/setters.

Why Encapsulate?

  • Data Protection — External code can't accidentally corrupt your object's state
  • Validation — You can validate data inside setters before accepting it
  • Flexibility — You can change internal implementation without breaking external code
  • Maintainability — Bugs are easier to find when data changes happen in one place
public class BankAccount {
    private double balance;   // hidden from outside

    public double getBalance() {
        return balance;
    }

    public void deposit(double amount) {
        if (amount > 0) {          // validation in setter
            balance += amount;
        }
    }

    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        } else {
            throw new IllegalArgumentException("Invalid amount");
        }
    }
}
A well-encapsulated class is like a vending machine — you put in a request, get the result, but have no access to the internal mechanism.

One-Line AnswerOverloading = same method name, different parameters, in same class (compile-time). Overriding = same method signature, in child class (runtime).

Always use @Override annotation when overriding. It lets the compiler catch mistakes if the signature doesn't match the parent method.
Q 12 What is a Constructor? What are its types? ▼
One-Line AnswerA constructor is a special method with the same name as the class and no return type, called automatically when an object is created to initialize its state.

Three Types of Constructors

class Person {
    String name;
    int age;

    // 1. Default Constructor (no parameters)
    Person() {
        name = "Unknown";
        age = 0;
    }

    // 2. Parameterized Constructor
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 3. Copy Constructor
    Person(Person other) {
        this.name = other.name;
        this.age  = other.age;
    }
}

Important Rules

  • Same name as the class, no return type (not even void)
  • If you don't define one, Java provides a free default constructor
  • If you define a parameterized constructor, Java no longer provides the default — you must write it yourself if needed
  • Constructors can be overloaded but cannot be overridden
  • Use this() to call another constructor in the same class (constructor chaining)
Constructor chaining with this() must be the FIRST statement in the constructor body.

Java Keywords

this, super, static, final and their real-world usage

Q 13 What is the this keyword in Java? ▼
One-Line Answer'this' refers to the current object instance. Used to resolve name conflicts between instance variables and parameters, call other constructors, and pass the current object.

4 Uses of this

class Person {
    private String name;
    private int age;

    // USE 1: Differentiate instance var from parameter
    Person(String name, int age) {
        this.name = name;  // this.name = instance variable
        this.age  = age;
    }

    // USE 2: Constructor chaining — call another constructor
    Person(String name) {
        this(name, 0);  // calls Person(String, int)
    }

    // USE 3: Pass current object to another method
    void register() {
        Registry.add(this);  // passes this Person object
    }

    // USE 4: Return current object (method chaining / builder pattern)
    Person setName(String name) {
        this.name = name;
        return this;  // enables chaining: person.setName("X").setAge(20)
    }
}
The 'return this' pattern is the foundation of the Builder Pattern and fluent APIs — very common in interview discussions!
Q 14 What is the super keyword in Java? ▼
One-Line Answer'super' refers to the parent class. Used to call the parent constructor, access overridden parent methods, and access hidden parent fields.
class Animal {
    String type = "Animal";
    void sound() { System.out.println("Generic sound"); }
}

class Dog extends Animal {
    String type = "Dog";  // hides parent field

    Dog() {
        super();  // MUST be first — calls Animal() constructor
    }

    @Override
    void sound() {
        super.sound();  // calls Animal's sound() first
        System.out.println("Woof!");
    }

    void printTypes() {
        System.out.println(super.type);  // "Animal"
        System.out.println(this.type);   // "Dog"
    }
}
super() must be the FIRST statement in a constructor. If you don't write it, Java inserts super() automatically (calling the parent's no-arg constructor).
Q 15 What is the static keyword? What can be static in Java? ▼
One-Line Answerstatic members belong to the class itself, not to any specific object instance. They are shared across all objects and can be accessed without creating an object.

What Can Be Static?

Static ThingDescriptionExample Use Case
Static variableShared across all instancesCounter of objects created
Static methodCalled on class, not objectMath.max(), Collections.sort()
Static blockRuns once when class loadsLoading config, registering drivers
Static nested classNested class without outer referenceBuilder pattern helper classes
class Counter {
    static int count = 0;    // shared across all Counter objects

    static {
        System.out.println("Class loaded!");  // static block
    }

    Counter() { count++; }  // increments on each new object

    static void showCount() {
        System.out.println("Total: " + count);
    }
}
// No object needed:
Counter.showCount();  // "Total: 0"

Important Restrictions

  • Static methods cannot access instance (non-static) variables or methods
  • Static methods cannot use this or super keywords
  • Static variables are stored in Metaspace (not heap)
Avoid overusing static — it makes testing harder and couples code. Use it for true utility methods (like Math class) or constants.
Q 16 What is the final keyword in Java? ▼
One-Line Answerfinal prevents modification: final variable = constant, final method = cannot be overridden, final class = cannot be subclassed.
Applied ToEffectExample
VariableValue cannot change (constant)final int MAX = 100;
MethodCannot be overridden in subclasspublic final void lock()
ClassCannot be extended/subclassedpublic final class String
// final variable: value locked after assignment
final int MAX_SIZE = 100;
// MAX_SIZE = 200; → Compile ERROR

// final reference: reference locked, but object content can change!
final List<String> list = new ArrayList<>();
list.add("Java");  // ✅ OK — modifying content
// list = new ArrayList<>(); → Compile ERROR — can't change reference
A common interview trap: a final reference variable can't be reassigned, but the object it points to can still be mutated. True immutability requires more than just final!

Strings in Java

Immutability, String pool, and the three String classes

Q 17 Why is String immutable in Java? ▼
One-Line AnswerString is immutable because once created, its value cannot be changed — any "modification" creates a new String object. This enables security, thread safety, and String pool optimization.

Proof of Immutability

String s1 = "Hello";
String s2 = s1;        // s2 points to same object
s1 = s1 + " World";   // creates NEW object "Hello World"

System.out.println(s1);  // "Hello World" (new object)
System.out.println(s2);  // "Hello"       (original unchanged)

4 Reasons Java Made String Immutable

  • Security — File paths, DB URLs, passwords are Strings. Immutability prevents malicious code from changing them after validation.
  • String Pool — Multiple references can safely share the same String object in the pool, saving memory.
  • Thread Safety — Immutable objects can be shared between threads without synchronization.
  • Hashcode Caching — String's hashcode is cached after first computation. Since value can't change, the cached hash is always valid (crucial for use as HashMap keys).
String pool: String a = "Java"; String b = "Java"; — both point to the same object! But new String("Java") always creates a new heap object.
Q 18 What is the difference between String, StringBuffer, and StringBuilder? ▼
One-Line AnswerString is immutable. StringBuffer is mutable and thread-safe (synchronized). StringBuilder is mutable, not thread-safe, but fastest.
FeatureStringStringBufferStringBuilder
Mutable?❌ Immutable✅ Mutable✅ Mutable
Thread-safe?✅ Yes✅ Yes (synchronized)❌ No
PerformanceSlow for concatModerateFastest
When to useConstants, short stringsMulti-threaded string buildingSingle-threaded string building

// String — creates new object every time

Q 24 What is the difference between Comparable and Comparator? ▼
One-Line AnswerComparable defines the natural/default sort order inside the class (compareTo). Comparator defines custom sort logic outside the class (compare) — allowing multiple sort strategies.
FeatureComparableComparator
Packagejava.langjava.util
MethodcompareTo(T obj)compare(T o1, T o2)
LocationInside the classSeparate class or lambda
Sort typesOne (natural order)Multiple custom orders

// Comparable — sort by age naturally
class Student implements Comparable<Student> {
    int age; String name;
    public int compareTo(Student other) { return this.age - other.age; }
}

// Comparator — sort by name (external)
Collections.sort(students, (Comparator.comparing(Student::getName)));
Collections.sort(students, (Comparator.comparingInt(s -> s.age)));
Use Comparable for natural ordering (String sorts alphabetically, Integer sorts numerically). Use Comparator when you need custom or multiple sort strategies.
Q 25 What is Optional in Java? Why use it? ▼
One-Line AnswerOptional is a container introduced in Java 8 that may or may not hold a value — it's a better way to handle null, preventing NullPointerExceptions by making null handling explicit.
// Without Optional — NPE risk
String name = getUserName(id);  // might be null!
int len = name.length();       // BOOM — NullPointerException

// With Optional — safe, explicit
Optional<String> name = Optional.ofNullable(getUserName(id));

// Method 1: orElse — return default if empty
String display = name.orElse("Anonymous");

// Method 2: ifPresent — run only if value exists
name.ifPresent(n -> System.out.println(n.toUpperCase()));

// Method 3: map — transform the value
Optional<String> upper = name.map(String::toUpperCase);

// Method 4: orElseThrow — throw exception if empty
String required = name.orElseThrow(() -> new RuntimeException("Name required"));

Creating Optional

  • Optional.of(value) — throws NPE if value is null
  • Optional.ofNullable(value) — safe, handles null
  • Optional.empty() — explicitly empty container
Don't use Optional as method parameters or class fields — only as return types. It's designed to signal "this method might not return a value."

Java Advanced Concepts

JVM internals, memory, GC, and Java 8+ features

Q 26 How does Java work internally? (Compilation to Execution) ▼
One-Line AnswerJava source code (.java) is compiled to bytecode (.class) by javac, then the JVM loads, verifies, and executes it — using the JIT compiler to convert hot code to native machine code.

Step-by-Step Execution Flow

  • Step 1 — Write: Developer writes Hello.java
  • Step 2 — Compile: javac Hello.java → produces Hello.class (bytecode)
  • Step 3 — ClassLoader: JVM's ClassLoader loads the .class file into memory
  • Step 4 — Bytecode Verifier: Checks bytecode for security violations and type safety
  • Step 5 — Interpreter: JVM interprets bytecode line-by-line (slower start)
  • Step 6 — JIT Compiler: Frequently-called methods ("hot" code) are compiled to native machine code — making them run fast
  • Step 7 — Execution: Program runs with GC managing memory automatically
Bytecode is the secret to WORA. It's not machine code (CPU-specific), it's JVM code — any JVM on any OS can run it.
Q 27 What are the memory areas in JVM? ▼
One-Line AnswerJVM has Heap (objects, GC-managed), Stack (method calls + local vars, per-thread), Metaspace (class metadata), PC Register, and Native Method Stack.

Common Memory Errors

  • OutOfMemoryError: Heap is full — too many objects alive (fix: tune heap size, fix memory leaks)
  • StackOverflowError: Stack is full — usually from infinite recursion
Metaspace replaced PermGen in Java 8. PermGen had fixed size and caused OutOfMemoryError. Metaspace uses native memory and grows dynamically.
Q 28 What is Garbage Collection? How does it work? ▼
One-Line AnswerGC automatically reclaims heap memory from objects that are no longer reachable/referenced — freeing developers from manual memory management.

How Objects Become Garbage

void method() {
    String s = new String("Hello");  // object created on heap
    s = null;                      // no references → eligible for GC
}                                  // when method returns, s goes out of scope

GC Types

  • Minor GC — Cleans the Young Generation (Eden + Survivor spaces). Fast and frequent.
  • Major GC — Cleans the Old Generation. Slower.
  • Full GC — Cleans entire heap. Expensive — causes "stop-the-world" pause.

Q 30 What is try-with-resources? ▼
One-Line AnswerTry-with-resources (Java 7+) automatically closes resources (files, connections, streams) when the try block finishes — even if an exception occurs — eliminating resource leaks.
// OLD WAY — manual cleanup, error-prone
FileReader fr = null;
try {
    fr = new FileReader("file.txt");
    // use fr...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fr != null) fr.close();  // what if close() also throws?
}

// NEW WAY — automatic, safe, clean
try (FileReader fr = new FileReader("file.txt")) {
    // use fr...
}  // fr.close() called automatically!

Any class implementing AutoCloseable or Closeable can be used in try-with-resources.

Always use try-with-resources for files, database connections, sockets, and any I/O. It prevents resource leaks — a common source of production bugs.

Exception Handling

Checked, unchecked, try-catch-finally, and best practices

Q 31 What is Exception Handling in Java? ▼
One-Line AnswerException handling manages runtime errors gracefully using try-catch-finally. Checked exceptions must be handled at compile time; unchecked exceptions are optional to handle.

Exception Hierarchy

  • Throwable (root)
    • Error — Serious JVM problems, don't catch: OutOfMemoryError, StackOverflowError
    • Exception
      • Checked — Must handle or declare: IOException, SQLException
      • Unchecked (RuntimeException) — Optional: NullPointerException, ArrayIndexOutOfBoundsException

Complete Example

public void readFile(String path) throws IOException {
    Path baseDir = Paths.get("/safe/dir");
    Path resolvedPath = baseDir.resolve(path).normalize();

    // Validate path to prevent Path Traversal
    if (!resolvedPath.startsWith(baseDir)) {
        throw new SecurityException("Invalid path: " + path);
    }

    try (BufferedReader br = new BufferedReader(new FileReader(resolvedPath.toFile()))) {
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
    } catch (FileNotFoundException e) {
        // Specific exception first (more specific → less specific)
        System.err.println("File not found: " + e.getMessage());
    } catch (IOException e) {
        System.err.println("IO Error: " + e.getMessage());
    } finally {
        System.out.println("This ALWAYS runs");
    }
}

// Custom exception
class InsufficientFundsException extends RuntimeException {
    InsufficientFundsException(String message) { super(message); }
}

Keywords Summary

KeywordPurpose
tryWraps code that might throw an exception
catchHandles the exception
finallyAlways executes — even if exception occurs or return is used
throwExplicitly throw an exception: throw new IOException("error")
throwsDeclares that a method might throw: void method() throws IOException
Catch specific exceptions before general ones. finally always runs — use it for cleanup. Create custom exceptions for domain-specific errors.

Multi-threading & Concurrency

Threads, synchronization, deadlocks, and thread pools

Q 32 What is Multi-threading in Java? ▼
One-Line AnswerMulti-threading allows a program to run multiple threads concurrently, sharing the same process memory — enabling parallelism, better CPU utilization, and responsive applications.

Key Concepts

TermDefinition
ProcessA running program with its own memory space
ThreadSmallest execution unit within a process — shares process memory
ConcurrencyMultiple tasks making progress (possibly on 1 CPU via context switching)
ParallelismMultiple tasks running simultaneously on multiple CPU cores

Thread Lifecycle States

  • NEW — Thread created, not started yet
  • RUNNABLE — Ready to run, waiting for CPU
  • RUNNING — Actively executing on CPU
  • BLOCKED/WAITING — Waiting for lock or event
  • TERMINATED — Execution completed
Real-world analogy: A restaurant kitchen is one process. Multiple chefs (threads) work concurrently — one grills, one chops, one plates — sharing the same kitchen resources.
Q 33 How to create threads in Java? What are the ways? ▼
One-Line AnswerThree ways: extend Thread class, implement Runnable interface, or use Lambda (Java 8+). For production, always use ExecutorService/thread pools.
// WAY 1: Extend Thread class
class MyThread extends Thread {
    public void run() { System.out.println("Thread 1 running"); }
}
new MyThread().start(); // NEVER call run() directly!

// WAY 2: Implement Runnable (RECOMMENDED — composition over inheritance)
class MyTask implements Runnable {
    public void run() { System.out.println("Runnable running"); }
}
new Thread(new MyTask()).start();

// WAY 3: Lambda (Java 8+)
new Thread(() -> System.out.println("Lambda thread")).start();

// WAY 4: ExecutorService (PRODUCTION BEST PRACTICE)
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> System.out.println("Pool thread"));
pool.shutdown();
Prefer Runnable over extending Thread (you can still extend another class). Use ExecutorService in production — raw threads are unmanaged and wasteful.

Q 35 What is Synchronization? What are thread safety issues? ▼
One-Line AnswerSynchronization ensures only one thread accesses a critical section at a time, preventing race conditions on shared mutable state.
// PROBLEM: Race condition — count++ is NOT atomic!
class Counter {
    private int count = 0;
    public void increment() { count++; }  // BUG: read-modify-write = 3 steps!
}

// SOLUTION 1: synchronized method
public synchronized void increment() { count++; }

// SOLUTION 2: synchronized block (more granular)
public void increment() {
    synchronized(this) { count++; }
}

// SOLUTION 3: AtomicInteger (best for simple counters)
private AtomicInteger count = new AtomicInteger(0);
public void increment() { count.incrementAndGet(); }

Thread Safety Issues

ProblemDescriptionSolution
Race ConditionMultiple threads modify shared data — final value unpredictablesynchronized, AtomicInteger
DeadlockTwo threads wait for each other's lock — both frozen foreverLock ordering, tryLock with timeout
StarvationThread never gets CPU/lock timeFair locks, thread priority tuning
LivelockThreads active but constantly reacting to each other, no progressIntroduce randomness in retry
Use AtomicInteger/AtomicLong for simple counters. For complex scenarios, prefer java.util.concurrent (ReentrantLock, CopyOnWriteArrayList, ConcurrentHashMap) over raw synchronized.

Q 38 What is a Functional Interface in Java? Added ▼
One-Line AnswerA functional interface has exactly ONE abstract method. It can be used with lambda expressions. Common examples: Runnable, Comparator, Predicate, Function, Consumer, Supplier.

Q 39 What is the Stream API in Java? Added ▼

Q 41 What is a deadlock? How do you prevent it? Added ▼
One-Line AnswerDeadlock is when two or more threads are waiting for each other to release a lock — permanently blocked, neither can proceed.

Classic Deadlock Scenario

// Thread 1: holds Lock A, waits for Lock B
// Thread 2: holds Lock B, waits for Lock A
// → Both frozen forever!

synchronized(lockA) {
    synchronized(lockB) { // Thread 1 order: A → B
        // work
    }
}

synchronized(lockB) {
    synchronized(lockA) { // Thread 2 order: B → A  ← DEADLOCK!
        // work
    }
}

Deadlock Prevention Strategies

  • Lock ordering — Always acquire locks in the same order across all threads
  • tryLock with timeout — Use ReentrantLock.tryLock(timeout) instead of blocking indefinitely
  • Lock timeout detection — Monitor threads and break deadlock if detected
  • Avoid nested locks — Minimize scenarios where one lock is held while acquiring another
  • Use higher-level concurrency — java.util.concurrent classes are designed to avoid deadlocks
Deadlock has 4 necessary conditions (Coffman's conditions): Mutual Exclusion, Hold and Wait, No Preemption, Circular Wait. Removing any one condition prevents deadlock.

Q 44 What is the volatile keyword? Added ▼

Q 45 What is the difference between == and equals() for Integer wrapper? Added ▼
One-Line AnswerJava caches Integer objects from -128 to 127 in a pool. So == works for these values but fails for larger ones. Always use equals() for Integer comparison.
Integer a = 100;
Integer b = 100;
System.out.println(a == b);      // true  ← from Integer cache
System.out.println(a.equals(b)); // true

Integer x = 200;
Integer y = 200;
System.out.println(x == y);      // false ← outside cache range!
System.out.println(x.equals(y)); // true  ← always use this!
This is a classic Java interview trick question! Always use equals() for all object comparisons. The Integer cache only covers -128 to 127.
Q 46 What is Dependency Injection? How does Spring use it? Added ▼
One-Line AnswerDependency Injection (DI) is a design pattern where an object receives its dependencies from outside (injected) rather than creating them itself — enabling loose coupling and testability.

Without DI (tight coupling)

class OrderService {
    private PaymentService payment = new PaymentService(); // hardcoded!
}

With DI (loose coupling)

@Service
class OrderService {
    private final PaymentService payment;

    @Autowired  // Spring injects PaymentService
    OrderService(PaymentService payment) {
        this.payment = payment;
    }
}

Three Types of Injection in Spring

  • Constructor Injection — ✅ Recommended (dependencies immutable, easier to test)
  • Setter Injection — For optional dependencies
  • Field Injection (@Autowired on field) — ❌ Avoid (hides dependencies, hard to test)
Always prefer constructor injection. It makes dependencies explicit, supports immutability (final fields), and makes unit testing without Spring container easy.
